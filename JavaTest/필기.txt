Hello World!

1. 메모장
	- 코드 작성
	- 산출물 > Hello.java > Source Code File
	- 소스 파일은 프로그램이 아니다. > 텍스트 파일이다. 
	- 프로그램밍 코드를 작성 > 프로그램 완성 > CPU에게 명령어 전달할 목적으로 작성
	- 어떤 언어로 구성? > 자바(x) > 사람이 쓰는 언어로 구성
	소스 파일의 명령어의 실행 주체 > 컴퓨터(CPU) > 이진데이터인 1, 0만 인식
	
2. javac.exe Hello.java
	- 1차 컴파일
	- javac.exe 파일명
	- javac.exe > Java Compiler > 사람이 작성한 코드를 컴퓨터가 이해하는 코드로 번역시켜주는 프로그램
	- 번역 작업 실행 > Compile
	- 산출물 > Hello.class > 실행 파일(클래스 파일)
	- Hello.class > 컴퓨터 실행(바이너리 코드, 이진 데이터, 기계어, Maching code)
	- 중간언어(IL) 산출물 > 실행 파일 (=프로그램)
	
3. java.exe Hello
	- 2차 컴파일, 인터프리터, JIT(Just In Time) 컴파일러
	- java.exe > Java Compiler, Java Interpreter, 실행기
	- java.exe 클래스명 > 최종 실행
	- 최종적으로 운영체제에 적합한 기계어로 바꿔주는 번역 작업
	
* 인터프리터
소스 -> 번역 -> 기계어

1. 컴파일 > 컴파일러 > 산출물 발생 > 초기 실행 비쌈 + 반복 실행 쌈
: 번역 - 한글로 된 책을 번역가를 시켜서 영어로 번역하여 미국에 사는 친구에게 보내는 방식
	- 단점 : 시간이 오래 걸린다.
	- 장점 : 번역을 한번만 하면 끝이다.

2. 인터프리트 > 인터프리터 > 산출물 없음 > 초기 실행 중간 + 반복 실행 중간
: 통역 - 한글로 된 책을 영어로 읽기 쉬우라고 번역하는 경우 통역가를 데리고와서 친구 옆에 붙여두고 한글로 된 책을 읽을 때마다 통역가가 실시간으로 통역해주는 것
	- 장점 :시간이 안 걸린다.
	- 단점 : 번역을 매번 해야 한다.
	
** 자바는 컴파일 방식으로 번역한다 !!
   SQL은 두 방식 모두 존재한다.
   CSS, JavaScript는 인터프리터 방식으로 번역한다.
